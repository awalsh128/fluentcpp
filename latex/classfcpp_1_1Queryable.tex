\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class fcpp\+::\+Queryable$<$ T $>$}

Core object used to query items and hold the sequence state. 

Any operation performed from \mbox{\hyperlink{classfcpp_1_1Queryable_aba673d44f38f19624c929b0a6ce441a1}{Queryable$<$\+T$>$}} will be moved into the new instance of \mbox{\hyperlink{classfcpp_1_1Queryable_aba673d44f38f19624c929b0a6ce441a1}{Queryable$<$\+T$>$}}. This means that any reference to the previous instance of \mbox{\hyperlink{classfcpp_1_1Queryable_aba673d44f38f19624c929b0a6ce441a1}{Queryable$<$\+T$>$}} may have unexpected behavior dependent on the operation performed and memory semantics of the object under query.

The nomenclature used in the documentation is as follows\+:
\begin{DoxyItemize}
\item Sequence refers to an ordered collection that can contain duplicates.
\item Item refers to an object in a sequence. It doesn\textquotesingle{}t have any inherit properties.
\item Key refers to an object that has equality.
\item Value refers to an object that has comparison. It the context of hash tables, it refers to the value part of the key-\/value pair.
\end{DoxyItemize}

\begin{DoxyRemark}{Remarks}
Note that not all methods are type dependent and will not always be valid. For example the \mbox{\hyperlink{classfcpp_1_1Queryable_acf826943a9e6e7d663facc280ec0185f}{max}} method will not work on items that don\textquotesingle{}t have comparability. This will show as an error at compile time though.
\end{DoxyRemark}
In terms of memory semantics, if a signature takes by value and not reference, it is because it makes modifications to the value and doesn\textquotesingle{}t want to cause side effects. To prevent a copy of a single use variable, it must be moved.

Lambdas can be passed with arguments by reference or value depending on the caller\textquotesingle{}s preference.
